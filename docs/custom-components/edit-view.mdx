The Edit View is where users interact with individual [Collection](../collections/overview) and [Global](../globals/overview) Documents within the [Admin Panel](../admin/overview). This is where they can view, edit, and save their content.

The Edit View contains the form in which submits the data to the server. It also contains buttons for saving, publishing, and previewing the document, all of which can be customized.

<Banner type="info">
  **Note:**
  The Edit View is one of many [Document Views](./custom-views#document-views) in the Payload Admin Panel. Each Document View is responsible for a different aspect of the interacting with Documents. The Edit View is specifically designed for editing and saving Document data.
</Banner>

## Custom Edit View

To swap out the entire Edit View with a [Custom View](./custom-views), use the `admin.components.views.edit` property in your [Payload Config](../configuration/overview):

```tsx
import { buildConfig } from 'payload'

const config = buildConfig({
  // ...
  admin: {
    components: {
      views: {
        // highlight-start
        edit: '/path/to/MyCustomEditView',
        // highlight-end
      },
    },
  },
})
```

Here's are some examples of simple Edit View components. These can be either Server or Client components, depending on your needs. Keep in mind that the props and types for these components are different depending on its type.

#### Server Component

```tsx
import React from 'react'

import type { ServerSideEditViewProps } from 'payload'

import { DefaultEditView } from '@payloadcms/ui'

const MyCustomServerEditView: React.FC<ServerSideEditViewProps> = ({
  Description,
  PreviewButton,
  PublishButton,
  SaveButton,
  SaveDraftButton,
  Upload
}) => {
  return (
    <DefaultEditView
      Description={Description}
      PreviewButton={PreviewButton}
      PublishButton={PublishButton}
      SaveButton={SaveButton}
      SaveDraftButton={SaveDraftButton}
      Upload={Upload}
    />
  )
}
```

_For details on how to build Custom Views, including all available props, see [Building Custom Views](./custom-views#building-custom-views)._

#### Client Component

```tsx
'use client'
import React from 'react'

import type { ClientSideEditViewProps } from 'payload'

import { DefaultEditView } from '@payloadcms/ui'

const MyCustomClientEditView: React.FC<ClientSideEditViewProps> = (props) => {
  return (
    <DefaultEditView {...props} />
  )
}
```

_For details on how to build Custom Views, including all available props, see [Building Custom Views](./custom-views#building-custom-views)._

## Custom Components

In addition to swapping out the entire Edit View with a [Custom View](./custom-views), you can also override individual components. This allows you to customize specific parts of the Edit View without swapping out the entire view for your own.

<Banner type="warning">
  **Note:**
  Collection and Globals are keyed to a different property in the `admin.components` object have slightly different options. Be sure to use the correct key for the type of document you are working with.
</Banner>

#### Collections

To override Edit View components for a Collection, use the `admin.components.edit` property in your [Collection Config](../configuration/collections):

```ts
import type { CollectionConfig } from 'payload'

export const MyCollection: CollectionConfig = {
  // ...
  admin: {
    components: {
      // highlight-start
      edit: {
        // ...
      },
      // highlight-end
    },
  },
}
```

The following options are available:

| Path                  | Description                                                                                                                                                                                                         |
|-----------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `SaveButton`         | A button that saves the current document. [More details](#SaveButton).                                                                                                                                  |
| `SaveDraftButton`    | A button that saves the current document as a draft. [More details](#SaveDraftButton).                                                                                                                                  |
| `PublishButton`      | A button that publishes the current document. [More details](#PublishButton).                                                                                                                                  |
| `PreviewButton`      | A button that previews the current document. [More details](#PreviewButton).                                                                                                                                  |
| `Upload`             | A file upload component. [More details](#Upload).                                                                                                                                  |

#### Globals

To override Edit View components for Globals, use the `admin.components.elements` property in your [Global Config](../configuration/globals):

```ts
import type { CollectionConfig } from 'payload'

export const MyCollection: CollectionConfig = {
  // ...
  admin: {
    components: {
      elements: { // highlight-line
        // ...
      },
    },
  },
}
```

The following options are available:

| Path                  | Description                                                                                                                                                                                                         |
|-----------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `SaveButton`         | A button that saves the current document. [More details](#SaveButton).                                                                                                                                  |
| `SaveDraftButton`    | A button that saves the current document as a draft. [More details](#SaveDraftButton).                                                                                                                                  |
| `PublishButton`      | A button that publishes the current document. [More details](#PublishButton).                                                                                                                                  |
| `PreviewButton`      | A button that previews the current document. [More details](#PreviewButton).                                                                                                                                  |

### SaveButton

```ts
import type { CollectionConfig } from 'payload'

export const MyCollection: CollectionConfig = {
  // ...
  admin: {
    components: {
      // highlight-start
      afterListTable: [
        '/path/to/MySaveButtonComponent',
      ],
      // highlight-end
    },
  },
}
```

Here's an example of a simple `SaveButton` component:

```tsx
import React from 'react'
import { SaveButton } from '@payloadcms/ui'

const MySaveButtonComponent: React.FC = () => {
  return (
    <SaveButton label="Save" />
  )
}
```
